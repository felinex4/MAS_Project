Optimisation Methodology

Model

Problem Formulation

Capacitated Vehicle Routing Problem (CVRP) — a classical Operations Research model. Given a set of locations each with a passenger demand, find the minimum-cost set of routes for a fixed fleet of buses, each with a seat limit.

Solver

Solver Engine

Google OR-Tools Routing API. Initial solution generated using the Path Cheapest Arc heuristic. This seed is then iteratively improved using Guided Local Search (GLS) metaheuristic within a 4-second time limit.

Distances

Real Road Distances

Straight-line (Haversine) distances are inaccurate for hilly Sri Lanka. All pairwise distances are fetched from the Open Source Routing Machine (OSRM) public API, using actual OpenStreetMap road network data.

Constraints

Time Constraints

A Distance Dimension is added to the solver that caps each bus route at a maximum total distance. This models a real-world commute time limit — workers must reach home within a reasonable duration.

Data Generation Method

Location

Location Generation

Random coordinates are generated within a configurable radius of the factory. Each coordinate is submitted to the OSRM Nearest API in parallel (40 concurrent threads) to snap it to the closest actual road, guaranteeing 100% routability.

Demand

Demand Generation

Passenger demand at each stop is generated using a Uniform Random Distribution between 5 and 25 passengers per stop per shift. This simulates day-to-day variation in workforce attendance. Seed 42 ensures reproducible results.

Workforce

Workforce Estimation

MAS Active Controline Pallekele operates two full shift cycles daily. Summing passenger demands across all 99 stops for both Collect shifts gives an estimated — unique employees who use the company transport per day (morning + afternoon shifts combined). Drop shifts carry the same people home, so they are not counted twice.
